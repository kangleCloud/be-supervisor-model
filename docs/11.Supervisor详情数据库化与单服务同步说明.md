# Supervisor 详情数据库化与单服务同步说明

## 背景与目标

详情页面需要快速打开，同时仍要支持按需读取远端真实现场，因此详情与同步必须拆分为两类接口。

## 当前问题或修改原因

如果打开详情就自动读取远端 `.ini`、`.bak` 和 `supervisorctl status`，页面会慢、不可达时也会影响基础浏览能力。

## 业务规则变更

- 详情接口只查数据库
- 同步接口才允许读取远端状态和配置
- 归档记录默认不允许执行详情同步

## 接口或配置变更

### 详情

`GET /admin/api/supervisor/services/{contentProgramName}`

返回数据库快照字段，包括：

- 基础标识
- 结构化配置字段
- 运行快照
- 归档状态
- 最近一次同步结果

### 同步

`POST /admin/api/supervisor/services/{contentProgramName}/sync`

同步内容包括：

- `status/pid/uptime`
- `command/directory/stdoutLogfile`
- `configContent`
- `backupConfigContent`
- `hasBackup`
- `lastSyncAt/syncStatus/syncError`

## 对调用方/运维的影响

- 前端打开详情时只调详情接口
- 用户主动点击“同步”后再刷新详情
- 运维可通过同步结果判断远端配置缺失、备份缺失或结构化字段解析失败

## 验证方式

- 详情接口不触发任何远端命令
- 同步接口可更新当前配置和备份配置
- 归档记录同步被拒绝
