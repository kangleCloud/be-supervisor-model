# Supervisor导入初始化API约定

## 背景与目标

当前前端如果要把远端现网 Supervisor 配置初始化到数据库，只能依赖运维手工执行脚本，缺少可直接接入的 HTTP 能力。

本次目标是提供一个前端可直接调用的初始化导入 API，同时保留现有脚本作为命令行兜底入口，并确保两者复用同一套业务逻辑。

## 当前问题或修改原因

- 前端无法直接触发远端配置初始化导入，页面联调和验收都依赖人工脚本。
- 旧脚本输出是命令行文本，不适合作为前端稳定契约。
- 导入预检与正式导入的返回格式没有统一约定，不利于前端先展示范围、再确认执行。
- 历史实现中，`host` 字段允许传入逻辑别名（如 `web-104`），实际通过 `ansiblePattern` 映射到 inventory 中的主机。这种别名机制导致前端必须感知后端 inventory 配置细节，且增加了调试复杂度。

## 业务规则变更

- 新增 `POST /admin/api/supervisor/imports` 作为前端专用初始化导入接口。
- 接口采用单接口双模式：
  - `mode=DRY_RUN`：只返回逐文件预检结果，不写 `sys_supervisor_service`
  - `mode=APPLY`：同步执行正式导入，逐文件写库并返回结果
- 每次只允许导入单台主机。
- API 固定递归扫描 `/etc/supervisord.d` 下全部 `*.ini`。
- API 固定排除 `.ini.bak` 和 `.ini.bak.*`。
- 导入过程只读远端文件并写数据库，不修改远端 `.ini`，也不执行任何 `supervisorctl reread/update/start/stop/restart`。
- 导入前置阶段保留 `hostname` 诊断探测，但该探测只输出调试信息，不参与成功失败判定。
- 如果同路径已有 `TEMPLATE_MANAGED` 记录，正式导入允许覆盖为 `IMPORTED_READONLY`，并在结果消息中明确提示。
- 如果同主机下 `programName` 已存在但 `configPath` 不同，该文件固定返回 `SKIPPED`，不影响其他文件继续执行。
- **引入按 IP 匹配远端主机约定**：
  - `host` 字段必须是白名单中的 IP 地址，不再支持逻辑别名。
  - `ansible` 执行器固定使用 `host.ip` 作为 inventory pattern，inventory 需保证可按该 IP 直接匹配。
  - 当 ansible inventory 中未找到该 IP 对应的主机时，接口直接返回失败，错误文案为"目标主机未匹配"。
  - 当 ansible/SSH 连接失败、远端主机 `UNREACHABLE`、连接超时或认证失败时，接口直接返回失败，错误文案统一为"目标主机不可达"。
  - 当目标主机远端 `/etc/supervisord.d` 下无任何 `*.ini` 文件时，接口直接返回失败，错误文案为"远端目录下无可用配置文件"。

## 接口或配置变更

### 请求

接口路径：

```text
POST /admin/api/supervisor/imports
```

请求体：

```json
{
  "host": "10.1.0.104",
  "mode": "DRY_RUN"
}
```

请求字段固定为：

- `host`：目标主机 IP
- `mode`：`DRY_RUN | APPLY`

### 响应

失败场景：

- ansible inventory 中未找到该 IP 时，返回 `404`，`msg` 为"目标主机未匹配"
- 目标主机 SSH 不可达、`UNREACHABLE`、连接超时或认证失败时，返回 `404`，`msg` 为"目标主机不可达"
- 目标主机远端 `/etc/supervisord.d` 下无任何 `*.ini` 文件时，返回 `404`，`msg` 为"远端目录下无可用配置文件"
- 仅当成功读取到至少一个 `*.ini` 文件时，才返回 `200` 成功响应
- 导入接口是同步长请求，多文件场景下总耗时可能超过默认 HTTP 代理超时。后端命令超时默认按 300s 配置（`supervisor.commandTimeoutSeconds` / `executor.timeoutSeconds`），可通过环境变量 `COMMAND_TIMEOUT_SECONDS` / `ANSIBLE_COMMAND_TIMEOUT_SECONDS` 覆盖

成功响应 `data` 固定为：

- `host`
- `mode`
- `summary`
- `items`

`summary` 固定包含：

- `planned`：通过预检、允许导入的文件数
- `imported`：本次新插入数据库的文件数
- `updated`：本次更新已有数据库记录的文件数
- `skipped`：本次跳过的文件数

`items` 每项固定包含：

- `configPath`
- `fileName`
- `contentProgramName`
- `programName`
- `configName`
- `jobName`
- `moduleName`
- `javaPath`
- `active`
- `port`
- `jarName`
- `xms`
- `xmx`
- `user`
- `manageMode`
- `metadataComplete`
- `parseWarnings`
- `result`
- `message`

`result` 枚举固定为：

- `PLANNED`
- `IMPORTED`
- `UPDATED`
- `SKIPPED`

示例响应：

```json
{
  "code": 200,
  "msg": "执行初始化导入成功",
  "data": {
    "host": "10.1.0.104",
    "mode": "DRY_RUN",
    "summary": {
      "planned": 2,
      "imported": 0,
      "updated": 0,
      "skipped": 1
    },
    "items": [
      {
        "configPath": "saas/legacy-name.ini",
        "fileName": "legacy-name.ini",
        "contentProgramName": "legacy_service",
        "programName": "legacy_service",
        "configName": "legacy-name.ini",
        "jobName": "legacy",
        "moduleName": "svc",
        "javaPath": "/usr/local/jdk17/bin/java",
        "active": "prod",
        "port": 9200,
        "jarName": "svc.jar",
        "xms": "128m",
        "xmx": "128m",
        "user": "root",
        "manageMode": "IMPORTED_READONLY",
        "metadataComplete": true,
        "parseWarnings": [],
        "result": "PLANNED",
        "message": "预检通过，正式导入时将新增记录"
      }
    ]
  }
}
```

### 前端接入约定

- “预检”按钮调用 `mode=DRY_RUN`。
- “确认导入”按钮调用 `mode=APPLY`。
- 前端应直接展示 `summary` 和 `items`，不要自行重算成功/失败数。
- 前端应按接口返回顺序展示 `items`，后端已经保证按 `configPath` 升序稳定返回。
- 如果 `items[].result=SKIPPED`，页面直接展示 `message` 作为跳过原因。
- 如果导入接口返回 404，说明出现了“inventory 未匹配 / 远端主机不可达 / 目录下无可用配置”中的一种，前端应展示错误文案，不再继续触发 APPLY。
- `host` 参数统一传递白名单 IP，不传逻辑别名。

## 对调用方/运维的影响

- 前端后续应优先使用初始化导入 API，不再依赖人工脚本作为主入口。
- 运维仍可保留 `scripts/import_supervisor_services.py` 作为命令行兜底手段。
- 因为 API 与脚本复用同一套服务层逻辑，预检结果、正式导入结果和冲突语义需要保持一致。

## 验证方式

- 运行 `python3 -m compileall app tests scripts`
- 运行 `.venv/bin/python -m pytest -q`
- 核心验收点：
  - `DRY_RUN` 只返回 `PLANNED/SKIPPED`，不新增 Supervisor 主数据
  - `APPLY` 返回 `IMPORTED/UPDATED/SKIPPED`，并正确写库
  - 同路径已有 `TEMPLATE_MANAGED` 记录时返回 `UPDATED`
  - 同 `programName` 不同路径冲突时只跳过该文件，不中断整次导入
  - 远端无可读 `*.ini` 文件时，API 返回 `404`
  - ansible inventory 中未匹配到目标 IP 时，API 返回 `404`
  - 远端主机 `UNREACHABLE`、SSH 失败或连接超时时，API 返回 `404`
