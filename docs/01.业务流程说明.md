# be-supervisor-model 业务流程说明

## 1. 背景与目标

`be-supervisor-model` 是一个面向运维后台的管理服务，不承担业务系统本身的业务逻辑。

当前系统只负责两类能力：

- 后台登录鉴权：提供账号登录、当前登录态查询、退出登录，并把登录令牌与登录审计写入 MySQL
- Supervisor 运维管理：管理目标主机上的 Supervisor 配置文件和进程状态

系统运行目标是为后续前端管理后台或内部运维工具提供统一的 HTTP 接口，用于：

- 限定白名单主机范围
- 安全生成和维护 `/etc/supervisord.d` 下的配置文件
- 执行 `supervisorctl` 常用命令
- 记录登录令牌和登录审计

## 2. 角色与职责

| 角色/组件 | 职责 |
| --- | --- |
| 运维管理员 | 通过后台登录后，查询和管理目标主机上的 Supervisor 服务 |
| `be-supervisor-model` | 统一处理登录鉴权、令牌校验、配置文件渲染、文件操作、端口冲突检查与 Supervisor 命令编排 |
| MySQL 8 | 存储用户信息、JWT 登录令牌和登录审计日志 |
| `local` 执行器 | 当管理服务与目标应用同机部署时，直接在本机执行文件操作和 `supervisorctl` |
| `ansible` 执行器 | 当目标主机为远程机器时，通过 `ansible` ad-hoc 命令执行受控操作 |
| 被管理主机 | 提供 `/etc/supervisord.d` 配置目录和 `supervisorctl` 命令环境 |

## 3. 认证与令牌流程

### 3.1 认证接口

认证接口统一挂载在：

```text
/admin/api/auth/*
```

当前提供：

- `POST /admin/api/auth/login`
- `GET /admin/api/auth/profile`
- `POST /admin/api/auth/logout`

### 3.2 登录链路

1. 运维管理员调用 `/admin/api/auth/login`，提交用户名和密码。
2. 服务从 `sys_user` 表读取账号信息，校验账号存在、未删除、状态为启用。
3. 服务使用密码校验逻辑比对明文密码和数据库中的哈希密码。
4. 校验通过后，服务签发 JWT 访问令牌。
5. 服务把本次登录令牌写入 `sys_login_token`，只保存 `token_jti` 和 `token_digest`，不把原始 token 明文落库。
6. 服务更新 `sys_user` 中的最近登录时间和登录地址。
7. 服务把本次成功或失败登录写入 `sys_login_log` 审计表。
8. 调用方后续访问受保护接口时，统一通过 `Authorization: Bearer <jwt>` 传递访问令牌。

### 3.3 令牌校验与退出

1. 受保护接口统一校验 Bearer JWT。
2. 服务先校验 JWT 签名和过期时间，再根据 `sub + jti` 到 `sys_login_token` 查询活动令牌。
3. 令牌存在且 `token_digest` 匹配时，才允许继续访问。
4. `/admin/api/auth/logout` 会把当前令牌写入 `revoked_time`，使同一 JWT 后续请求立即失效。

### 3.4 认证相关数据表

当前登录鉴权依赖以下表：

- `sys_user`：用户信息
- `sys_login_token`：JWT 登录令牌
- `sys_login_log`：登录审计日志
- `sys_schema_migration`：数据库迁移版本记录

数据库启动规则：

- 服务启动时自动创建业务库（若不存在）
- 自动执行 `app/database/migrations/*.sql`
- 账号不通过 HTTP 接口创建，运维需要手工维护 `sys_user`

## 4. Supervisor 管理流程

### 4.1 Supervisor 接口

Supervisor 管理接口统一挂载在：

```text
/admin/api/supervisor/*
```

当前提供：

- `GET /admin/api/supervisor/hosts`
- `GET /admin/api/supervisor/services`
- `GET /admin/api/supervisor/services/{programName}`
- `POST /admin/api/supervisor/services`
- `PUT /admin/api/supervisor/services/{programName}`
- `DELETE /admin/api/supervisor/services/{programName}`
- `POST /admin/api/supervisor/services/{programName}/start`
- `POST /admin/api/supervisor/services/{programName}/stop`
- `POST /admin/api/supervisor/services/{programName}/restart`
- `POST /admin/api/supervisor/services/{programName}/backup`
- `POST /admin/api/supervisor/services/{programName}/restore`
- `GET /admin/api/supervisor/ports/check`
- `POST /admin/api/supervisor/reread`
- `POST /admin/api/supervisor/update`
- `GET /admin/api/supervisor/status`

### 4.2 主机白名单与执行模型

目标主机必须来自配置中的 `hosts` 白名单，不允许调用方传入任意执行目标。

每台主机都绑定一种执行方式：

- `local`：直接在本机操作配置与执行命令
- `ansible`：通过 `ansible` 对远程主机执行受控文件与命令操作

当前执行器只暴露受控能力，不暴露任意 shell 执行接口，例如：

- 列出配置文件
- 读取配置文本
- 原子写入配置文件
- 复制、移动、删除文件
- 执行固定格式的 `supervisorctl` 命令

### 4.3 配置文件目录与命名规则

Supervisor 配置目录固定为：

```text
/etc/supervisord.d
```

只允许操作以下文件：

- `*.ini`
- `*.ini.bak`
- `*.ini.bak.*`

默认规则：

- `programName = {jobName}_{moduleName}`
- `configName = {programName}.ini`

新增或修改时，服务端负责：

- 校验 `jobName / moduleName / programName / configName` 安全正则
- 拒绝 `../`、绝对路径注入和备份文件名伪装
- 把配置文件路径限制在 `/etc/supervisord.d`

### 4.4 模板渲染规则

当前新增和修改接口允许调用方输入的主要字段为：

- `host`
- `jobName`
- `moduleName`
- `javaPath`
- `active`
- `port`
- `jarName`
- `configName`
- `xms`
- `xmx`
- `user`
- `autoStart`

其中 `autoStart` 只控制本次新增或修改完成后是否立即执行 `supervisorctl start`，不是模板配置项本身。

以下 Supervisor 模板参数固定为系统内建策略，不接受前端自定义输入：

- `autostart=true`
- `startsecs=10`
- `autorestart=true`
- `startretries=3`
- `priority=999`
- `redirect_stderr=true`
- `stdout_logfile_maxbytes=1GB`
- `stdout_logfile_backups=1`
- `stopasgroup=false`
- `killasgroup=false`

### 4.5 新增服务流程

1. 校验 Bearer JWT 与令牌状态。
2. 校验目标主机必须在白名单中。
3. 根据请求变量生成 `programName` 和 `configName`。
4. 检查目标配置文件是否已存在。
5. 扫描 `/etc/supervisord.d` 下的 `.ini/.ini.bak/.ini.bak.*`，检测端口冲突。
6. 按固定模板渲染 Supervisor 配置。
7. 原子写入目标配置文件。
8. 执行 `supervisorctl reread`。
9. 执行 `supervisorctl update`。
10. 如果 `autoStart=true`，再执行 `supervisorctl start {programName}`。

### 4.6 修改服务流程

1. 根据路径参数定位旧 `programName`。
2. 读取旧配置并停止原服务。
3. 自动备份旧配置为 `.bak`；若旧 `.bak` 已存在，先归档为时间戳文件。
4. 根据新请求重新生成 `programName` 和 `configName`，支持自动重命名。
5. 重新执行端口冲突检查，并排除当前配置文件。
6. 原子写入新配置。
7. 如果配置名发生变化，删除旧 `.ini` 主文件。
8. 执行 `supervisorctl reread` 和 `update`。
9. 如果 `autoStart=true`，启动新服务。

### 4.7 删除、备份、还原流程

删除流程：

1. 按 `programName` 找到目标配置。
2. 先执行 `supervisorctl stop`，允许“本来就未运行”的情况。
3. 自动备份当前 `.ini`。
4. 删除主配置文件。
5. 执行 `reread` 和 `update`。
6. 只有 `deleteBackup=true` 时才会删除对应 `.bak`。

备份流程：

1. 把 `*.ini` 复制为 `*.ini.bak`
2. 如果旧 `.bak` 已存在，先归档为 `*.ini.bak.YYYYMMDD-HHMMSS`

还原流程：

1. 定位 `*.ini.bak`
2. 如果当前 `.ini` 存在，先把当前 `.ini` 归档为时间戳文件
3. 用 `.bak` 复制还原出新的 `.ini`
4. 执行 `reread` 和 `update`

### 4.8 状态与端口冲突查询

服务列表和详情接口会合并两部分信息：

- 配置文件视角：配置名、原始内容、结构化变量
- 运行状态视角：`supervisorctl status` 返回结果

端口冲突检查会扫描：

- `*.ini`
- `*.ini.bak`
- `*.ini.bak.*`

匹配以下端口表达式：

- `-Dserver.port=xxxx`
- `server.port=xxxx`
- `port=xxxx`

## 5. 配置与运行模型

### 5.1 配置来源

当前配置采用：

- `config.yaml`
- 环境变量覆盖

环境变量优先级高于 YAML。

### 5.2 关键配置项

应用与认证相关：

- `APP_HOST`
- `APP_PORT`
- `APP_LOG_LEVEL`
- `DATABASE_HOST`
- `DATABASE_PORT`
- `DATABASE_NAME`
- `DATABASE_USER`
- `DATABASE_PASSWORD`
- `JWT_SECRET`
- `ACCESS_TOKEN_EXPIRE_MINUTES`

Supervisor 与执行器相关：

- `SUPERVISOR_CONF_DIR`
- `COMMAND_TIMEOUT_SECONDS`
- `EXECUTOR_TYPE`
- `ANSIBLE_INVENTORY_PATH`
- `ANSIBLE_REMOTE_USER`
- `ANSIBLE_COMMAND_TIMEOUT_SECONDS`
- `hosts[]`

## 6. 响应格式与错误边界

所有接口统一返回：

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

失败时同样保持 `code/msg/data` 结构，`msg` 一律使用中文。

当前主要错误码：

| HTTP 状态码 | 响应 code | 含义 |
| --- | --- | --- |
| `400` | `40000` | 参数错误 |
| `400` | `40001` | 非法主机 |
| `400` | `40002` | 非法配置名 |
| `401` | `40100` | 未登录、JWT 无效或会话已失效 |
| `404` | `40400` | 配置文件不存在 |
| `409` | `40003` | 端口冲突 |
| `409` | `40900` | 配置文件已存在 |
| `500` | `50000` | 内部错误 |
| `500` | `50010` | 远程命令执行失败 |
| `500` | `50020` | Supervisor 命令执行失败 |
| `500` | `50030` | 文件操作失败 |

## 7. 安全边界

系统当前安全边界如下：

- 所有 Supervisor 管理接口都必须先通过 Bearer JWT 校验
- JWT 不单独作为唯一凭证，必须同时命中服务端活动会话表
- 目标主机必须命中配置白名单
- 不允许任意命令执行
- 不允许路径穿越
- 所有配置文件必须限制在 `/etc/supervisord.d`
- `programName/jobName/moduleName/configName` 必须满足安全正则
- 端口必须在 `1-65535`
- token 明文不落库，只保存摘要
