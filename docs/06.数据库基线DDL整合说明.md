# 数据库基线 DDL 与迁移说明

## 背景与目标

数据库结构已经收敛到 Tortoise ORM + Aerich 管理，运行时不再执行历史 SQL migration。

## 当前问题或修改原因

如果同时保留“应用启动自动跑 SQL”和“Aerich 显式迁移”两条链路，会让 schema 来源不清晰，也会让旧库升级问题变得更隐蔽。

## 业务规则变更

- `migrations/models/` 是唯一 migration 来源
- `app/database/migrations/001_init_schema.sql` 仅保留为历史结构快照
- 应用启动只初始化数据库连接
- 运行前统一执行 `aerich upgrade`
- MySQL 启动时会做 schema fail-fast：
  - 必须存在 `config_path/file_name/content_program_name`
  - 不允许残留旧版 `program_name/config_name`
  - 不允许残留旧唯一键和旧索引

## 接口或配置变更

- 无新增 HTTP 接口
- 运维命令统一为：
  - `APP_ENV=dev .venv/bin/aerich upgrade`
  - `APP_ENV=dev .venv/bin/aerich migrate --name <message>`

## 对调用方/运维的影响

- 新库初始化和旧库升级都必须走 Aerich
- 若启动提示 schema 落后，应先升级数据库再拉起应用

## 验证方式

- 新库执行 `aerich upgrade` 后可正常启动
- 旧库残留旧字段时启动会直接失败并给出升级提示
