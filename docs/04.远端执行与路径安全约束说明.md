# 远端执行与路径安全约束说明

## 背景与目标

本专题用于说明远端执行边界、Ansible 主机匹配规则和 `/etc/supervisord.d` 路径安全约束。

## 当前问题或修改原因

系统已经支持 `ansible` 主机上的读取、创建、修改、运行操作、归档和删除；旧的“远端只读”说法已不符合现状，但路径安全约束仍然必须保留。

## 业务规则变更

- 目标主机必须来自白名单 `hosts`
- `ansible` 主机统一按 `host.ip` 作为 inventory pattern
- 远端操作能力包括：
  - 列表/详情同步读取
  - 初始化导入
  - 创建/修改
  - `start/stop/restart`
  - 归档/还原
  - 已归档记录硬删除
- 所有文件操作必须限制在 `/etc/supervisord.d`
- 只允许处理：
  - `*.ini`
  - `*.ini.bak`
  - `*.ini.bak.*`
- 拒绝路径穿越、绝对路径注入和非法文件名

## 接口或配置变更

- 远端能力由执行器自动决定，不新增独立权限配置
- `executor.inventoryPath` 必须存在且可读
- `executor.remoteUser` 和 `executor.timeoutSeconds` 由配置统一控制

## 对调用方/运维的影响

- 前端只需要传主机 IP，不再使用逻辑别名
- 运维必须保证 `/etc/ansible` 或自定义 inventory 可按主机 IP 直接匹配
- 所有“按配置路径操作”的接口都以数据库中的 `configPath` 为准

## 验证方式

- 白名单外主机被拒绝
- 非法路径和非法名称被拒绝
- `ansible` 主机可正常创建、归档和删除
- 远端 inventory 缺失或不可读时返回明确错误
