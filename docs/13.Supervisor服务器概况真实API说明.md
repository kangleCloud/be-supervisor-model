# Supervisor 服务器概况真实 API 说明

## 背景与目标

前端切换主机、手动刷新和刷新页面时，都需要拿到目标主机的真实概况，而不是 mock 数据。

## 当前问题或修改原因

概况接口天然是实时数据读取场景，不适合落库；同时远端不可达和本地不支持场景又必须能稳定回包，不能让前端用“0 值”误判成真实负载。

## 业务规则变更

- `GET /admin/api/supervisor/overview` 是唯一真实数据来源
- 不落库、不起后台线程、不做 mock 回填
- `local` 主机返回 `UNSUPPORTED`
- `ansible` 不可达时返回 `200 + UNREACHABLE`
- 不可达时数值字段继续返回 0，但前端必须结合：
  - `available`
  - `connectionState`
  - `checks`
  - `warnings`

## 接口或配置变更

- 概况采集复用 `executor.timeoutSeconds`
- 不再保留独立概况超时配置
- 响应头固定 `Cache-Control: no-store`

## 对调用方/运维的影响

- 前端不能回填 mock CPU、mock 内存或 mock 检查项
- 运维应优先关注 `warnings` 中的真实不可达原因，如 inventory、SSH 或远端命令失败

## 验证方式

- `local` 主机返回 `UNSUPPORTED`
- inventory 缺失时返回明确配置错误
- 远端 SSH 超时时返回 `UNREACHABLE`
