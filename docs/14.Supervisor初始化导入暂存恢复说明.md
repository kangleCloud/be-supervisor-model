# Supervisor 初始化导入暂存恢复说明

## 背景与目标

初始化导入常常需要较长预检时间，页面刷新、切换主机或重新打开弹窗后，必须能够恢复上一次结果。

## 当前问题或修改原因

如果预检完成后立刻清理暂存，前端在网络中断或页面刷新后就无法继续提交同一批数据。

## 业务规则变更

- 预检结果写入 `sys_supervisor_import_staging`
- 暂存按“当前用户 + 主机 IP”隔离
- 同一用户同一主机只保留最近一批
- `PRECHECK` 会覆盖旧暂存
- `COMMIT` 成功后才清理暂存
- `COMMIT` 失败时暂存保留

## 接口或配置变更

- `GET /admin/api/supervisor/imports/staging?host=...`
- `POST /admin/api/supervisor/imports` with `mode=PRECHECK`
- `POST /admin/api/supervisor/imports` with `mode=COMMIT`

`GET staging` 无数据时返回：

- `exists=false`
- 空 `summary`
- 空 `items`

## 对调用方/运维的影响

- 前端进入导入弹窗时应先调 `GET staging`
- 若 `exists=true`，前端直接展示已有结果，不应强制重新预检
- 运维脚本可继续复用同一套服务层逻辑

## 验证方式

- 预检后刷新页面仍能看到结果
- 同一用户重新预检后旧批次被覆盖
- 提交失败时暂存仍可继续查看和重试
