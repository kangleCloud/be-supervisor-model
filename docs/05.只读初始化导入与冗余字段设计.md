# 只读初始化导入与冗余字段设计

## 背景与目标

现网已有 Supervisor 配置时，需要先把远端现场快照导入数据库，再切换到“数据库为主”的列表与详情视图。

## 当前问题或修改原因

导入场景必须同时保留“真实文件路径”“真实文件名”“真实内容中的 program 名称”和“可解析的结构化字段”，否则无法准确表达 legacy 现场。

## 业务规则变更

- 导入只写数据库，不改远端现场
- `manage_mode` 仅分两类：
  - `TEMPLATE_MANAGED`
  - `IMPORTED_READONLY`
- 导入记录固定保留：
  - `config_path`
  - `file_name`
  - `content_program_name`
  - `baseline_content`
  - `metadata_complete`
  - `parse_warnings`
- `file_name` 与 `content_program_name` 永远独立保存
- 不能再用文件名反推 `content_program_name`
- `baseline_content` 代表模板基线或导入原文，不用于覆盖当前现场

## 接口或配置变更

- 导入接口扫描 `/etc/supervisord.d/**/*.ini`
- 导入响应返回逐文件 `summary + items`
- 导入记录可在后续被修改；一旦修改则转成 `TEMPLATE_MANAGED`

## 对调用方/运维的影响

- 前端展示导入结果时，应同时展示 `configPath/fileName/contentProgramName`
- 运维核对导入结果时，不能只看显示名称，必须同时校验现场路径和内容名称

## 验证方式

- 导入后数据库中能同时看到 `configPath/fileName/contentProgramName`
- legacy 配置文件名和内容名不一致时，数据不会被覆盖成同一个值
- 导入记录修改后 `manageMode` 正确变更
