# Supervisor 服务列表分页与数据库查询改造

## 背景与目标

服务列表需要稳定、可分页、可筛选，并避免每次进入页面都触发远端目录扫描或状态命令。

## 当前问题或修改原因

把列表建立在远端扫目录之上会导致性能不稳定，也会让前端查询操作和现场状态强耦合。

## 业务规则变更

- 列表接口固定只查数据库
- 不隐式执行：
  - `supervisorctl status`
  - 远端 `.ini/.bak` 读取
  - 目录扫描
- 默认只返回未归档记录
- 排序固定为 `update_time DESC, id DESC`

## 接口或配置变更

`GET /admin/api/supervisor/services`

查询参数：

- `host`
- `keyword`
- `status`
- `archived=false|true|all`
- `page`
- `pageSize`

关键字匹配字段：

- `content_program_name`
- `file_name`
- `config_path`
- `job_name`
- `module_name`
- `port`

## 对调用方/运维的影响

- 前端翻页、筛选、搜索都不再触发远端命令
- 若要刷新运行状态，必须显式调用状态刷新接口

## 验证方式

- 列表分页、状态过滤、关键字过滤正确
- monkeypatch 远端执行器后，列表接口仍可正常返回
