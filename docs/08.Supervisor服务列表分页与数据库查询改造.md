# Supervisor 服务列表分页与数据库查询改造

## 背景与目标

当前列表接口 `GET /admin/api/supervisor/services` 虽然主数据源已是 `sys_supervisor_service`，但每次查询仍会额外触发两类远端操作：
- `supervisorctl status` 补运行状态
- 按 `configPath` 读取远端文件补 `fileState`（旧实现）

当远端主机慢或文件多时，列表接口不再因远端操作而超时或失败。

## 当前问题或修改原因

- 列表接口无法支撑前端页面需要快速、稳定的数据展示需求
- 每查一次列表都要远端连接一次，导致小页面操作也要等几秒
- 没有分页能力，数据量大了以后前端无法按页加载

## 业务规则变更

### 1. 列表接口改为纯数据库分页查询

- 保持路径：`GET /admin/api/supervisor/services`
- 请求参数改为：
  - `host`：可选，目标主机 IP
  - `keyword`：可选，模糊匹配 `program_name / config_name / job_name / module_name / port`
  - `status`：可选，按数据库状态快照过滤
  - `page`：默认 `1`
  - `pageSize`：默认 `10`，只允许 `10 / 20 / 50`
- 响应结构改为分页对象：`records / page / pageSize / total / pages`
- 排序固定为 `update_time DESC, id DESC`
- 列表查询只查 `sys_supervisor_service`，完全禁止调用远端 executor 或 supervisorctl
- 详情接口保持现有实时行为，本次不做改造

### 2. 数据库补齐状态快照字段

`sys_supervisor_service` 新增列：

- `status VARCHAR(32) NOT NULL DEFAULT 'UNKNOWN'`
- `pid VARCHAR(32) DEFAULT NULL`
- `uptime VARCHAR(64) DEFAULT NULL`
- `status_sync_time DATETIME DEFAULT NULL`
- 新增索引：`(host_ip, status)`

### 3. 独立状态刷新接口

- 新增 `POST /admin/api/supervisor/services/status/refresh?host=10.1.0.104`
- 校验 JWT 和白名单主机
- 远端执行 `supervisorctl status`，解析 `programName / state / pid / uptime`
- 批量更新数据库 `status / pid / uptime / status_sync_time`
- 返回 `host / total / updated / missing`

### 4. 新增与导入的数据库状态默认值

- `POST /admin/api/supervisor/services` 成功后，落库 `status = 'UNKNOWN'`，不再实时查远端状态
- `POST /admin/api/supervisor/imports` `APPLY` 模式写库时，默认 `status = 'UNKNOWN'`

### 5. 新增状态枚举

- `RUNNING`
- `STOPPED`
- `FATAL`
- `BACKOFF`
- `STARTING`
- `STOPPING`
- `EXITED`
- `UNKNOWN`

### 6. 日志中文化

- 列表查询日志：`查询服务列表：目标主机=%s，关键字=%s，状态=%s，当前页=%s，每页条数=%s`
- 状态刷新日志：`刷新服务状态：目标主机=%s` / `刷新服务状态成功：目标主机=%s，更新条数=%s，未匹配条数=%s`

## 数据库变更

新增迁移 `002_add_supervisor_service_runtime_columns.sql`，用于补已有库的字段和索引。同时同步更新 `001_init_schema.sql` 中 `sys_supervisor_service` 的最终结构，新库初始化直接是最新形态。

## 对调用方/运维的影响

- 列表接口从数组改为分页对象，前端需要按新结构渲染
- 列表不再返回实时 `fileState`，不再返回实时 `status` 对象
- 新增或导入后如需准确运行状态，需调用独立刷新接口
- 详情接口不受本次改造影响，仍可实时读取远端状态
- 运行中的服务需重启后，新端点才生效

## 验证方式

- 运行 `python3 -m compileall app tests scripts`
- 运行 `.venv/bin/python -m pytest -q`（93 项测试全部通过）
- 核心验收点：
  - 列表默认 `page=1/pageSize=10`
  - `pageSize=20` / `pageSize=50` 正常工作
  - 非法 `pageSize` 返回请求参数非法
  - `status` 精确过滤
  - `keyword` 模糊匹配
  - 排序 `update_time DESC, id DESC`
  - 列表接口不触发任何远端命令
  - 状态刷新接口正确解析并落库
  - 新增服务后状态默认 `UNKNOWN`
